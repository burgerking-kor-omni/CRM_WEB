공통

  • - 스타일 가이드 목적 : 디자인팀 작업의 가이드를 제공하여 작업자마다 상이한 화면을 만들지 않고, 개발자들의 편의를 도모
  • - 예시화면과 html코드는 중요 코드나 클래스의 이해를 돕기 위함이므로 예시화면과 html 코드 상이할 수 있음
  • - 프로젝트에 따라 변경(디자인, 컬러값, 해상도 등..)될 수 있으며, 추가발생시 디자인팀에 서포트 요청할 것

기본환경

  • - 문서타입 : html5
    (단, IE8에서 html5를 지원하지 않기때문에 html5.js를 반드시 써준다)
  • - 기준 해상도 : 1280*n (세로스크롤 생성시 컨텐츠 width : 1260px, 실제로 컨텐트가 뿌려질 영역은 1220px정도. 간혹 1024인 프로젝트 존재)
  • - 브라우저 환경 : IE8.0 ~ , 크롬

css 구분 기준

  • 기본 위치 (베이스) : page/css
    • - base.css : 주요 태그의 리셋, 공통 클래스
    • - develop.css : 기존 베이스웹과 추후 버전 높이기 전 사이에 발생하는 공통 스타일 컨트롤
    • - layout.css : 화면의 레이아웃 잡는 용도
    • - style.css : 디자인 구현 및 각각의 컨트롤 (버튼, 테이블, 아이콘, jqgrid)
    • - main.css : 메인 포탈 화면. 프로젝트마다 유니크
    • - dashboard.css : 프로젝트에 따라 유,무가 결정되고, 프로젝트마다 유니크 함
  • 프로젝트에 따라 변경될 경우 (사이트) : page/css/site
    • - layout.css : (위를 기반으로 하여) 화면의 레이아웃 잡는 용도
    • - style.css : (위를 기반으로 하여) 디자인 구현 및 각각의 컨트롤, 프로젝트마다 추가되는 페이지 작업

화면 구조

ub-page > ub-frame > ub-layout > ub-control

  • - ub-page : 페이지를 구성하는 가장 기본 단위. document.body의 최상단 클래스명으로 정의한다.
  • - ub-frame : 페이지를 구성하는 커다란 레이아웃을 지칭하는 클래스. 유형에 따라 분류됨 (ex - top, contents)
  • - ub-layout : 컨텐츠 프레임에 표지되는 영역의 layout정의. 서브 클래스를 가질수 있다. (ex - search, list, button, paging..)
  • - ub-control : 페이지를 구성하는 요소. 서브 클래스를 가질수 있다. (ex - table, input, select, label..)

화면 레이아웃

컨텐츠의 목적에 따라 알맞은 레이아웃 지정을 통해 화면이 올바르게 나올 수 있도록 함

- Default : 기본 레이아웃이며, 가운데 정렬되는 형태.

HTML Code
								 < body class="ub-page default list"> 
										

											

												contents area
											

										

								 </body>

- Fullwidth : 양 사이드에 최소한의 여백을 가지며, 화면 사이즈에 맞게 컨텐츠가 100% 늘어나는 형태.

HTML Code
								<body class="ub-page fullwidth list">  
									

										

											contents area
										

									

								</body>

- 기타 레이아웃 : Popup, Innercontent, Tabcontent, Mainportal

화면 유형

  • 목록(list) - 주로 검색화면으로 구성
  • 상세(detail) - 주로 검색되어진 상세값을 볼 수있음
  • register 기타 등등..
같은 메타 템플릿을 사용하더라도, list이냐 detail이냐에 따라 다른 화면 나올수 있다. (ex - 검색 레이아웃 템플릿)

공통 클래스

  • - 폰트 왼쪽 정렬 : text-left
  • - 폰트 가운데 정렬 : text-center
  • - 폰트 오른쪽 정렬 : text-right
  • - 폼 태그 사용시, (수정불가)값만 뿌릴때 사용 : input-data
  • - 인풋 자동 완성 : input-auto
  • - 텍스트에 밑줄 긋기 : line

폰트 컬러 클래스

  • - 기본폰트 : 맑은고딕 12px #444444
  • - 포인트컬러 1 : 맑은고딕 12px #db482b / 클래스명 : txt-color01
  • - 포인트컬러 2 : 맑은고딕 12px #4b6aa9 / 클래스명 : txt-color02
  • - 링크컬러 1 : 맑은고딕 12px #db482b / 클래스명 : link-color01
  • - 링크컬러 2 : 맑은고딕 12px #4b6aa9 / 클래스명 : link-color02
포인트 컬러 사용시 메타에서 color에 font color=”red”로 사용하지 말고 위 포인트 컬러의 클래스를 사용한다.
일반적으로 label영역에 <span class="txt-color01">글씨</span> , 굵은 글씨일 경우 <b>태그 대신 <strong class="txt-color01">글씨</strong>를 사용하여 적용시켜준다.
2가지 이상 컬러 필요시 디자이너에게 요청할 것

공통 적용

  • - 날짜 (년-월-일) 부분 : 인풋 박스 사이즈 73px로 통일
  • - 기본 인풋박스 사이즈는 100px로 하되, 필요에 따라 유동적으로 한다.
  • - 현재 화면에 보이지 않는 부분은 확실하게 none처리 하여 브라우저에서 높이를 가지지 않게 처리한다.
  • - 테이블 셀의 경우 4칸일때 120px / 480px / 120px / n 으로 width값을 준다. (2칸일 경우 - 120px / n)
